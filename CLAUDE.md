# CV Bio Web Page — Agent Instructions

This project generates a bilingual (EN/RU) personal bio web page (SSG via Vite + Handlebars).

## File Structure

```

├── public/
│   └── assets/
│       ├── Tigran_Frangyan_Senior_Frontend_Developer.pdf
│       ├── avatar.png          ← optimized (160×160, generated by build)
│       ├── avatar.webp         ← optimized WebP (generated by build)
│       └── avatar-source.png  ← original hi-res (gitignored, kept locally)
├── src/
│   ├── index.html             ← Vite entry point (EN)
│   ├── ru/
│   │   └── index.html         ← Vite entry point (RU)

│   └── styles.css             ← All styles (themes, layout, animations, print)
├── scripts/

│       └── deploy.yml         ← GitHub Actions: build + deploy to GitHub Pages
├── build-meta.js              ← Per-language SEO/meta config (title, description, JSON-LD)

└── CLAUDE.md                  ← This file
```

## Architecture

All content is **pre-rendered at build time** via Handlebars. The client JS (`app.js`) handles only interactivity: theme toggle, language navigation, scroll animations, mobile sidebar, back-to-top.

## Build

```bash
npm run build
```

This runs `scripts/optimize-images.js` (avatar resize/compress) then `vite build`, generating:
- `dist/index.html` — English version
- `dist/ru/index.html` — Russian version

Deploy happens automatically via GitHub Actions on push to `main`.

**Dev server:**
```bash
npm run dev
```

## How to Update Content

1. **Use the `/pdf` skill** to extract text from the PDF in `public/assets/`.
2. **Edit `src/content-data.js`** — update the `CV_DATA` object (both `en` and `ru` keys).
3. **Edit `build-meta.js`** if the SEO title/description changes.
4. Run `npm run build` to verify, or check via `npm run dev`.

### CV_DATA structure

```js
export const CV_DATA = {
  en: {
    name, title, subtitle, tagline, summary,
    experience: [                              // array, most recent first
      { company, role, period, description, highlights: [...] }
    ],
    skills: {                                  // plain object, keys = category names
      "Core": [...],
    },
    education: { university, degree, period }, // single object, NOT array
    languages: [ { lang, level }, ... ],       // key is "lang", not "name"
    contacts: {                                // plain object, NOT array
      email: "...",
      github:   { label, url },
      linkedin: { label, url },
      telegram: { label, url }
    },
    nav: { about, experience, skills, education, contact },
    ui: { downloadCV, switchLang, language, darkMode, languages, getInTouch, contactCTA }
  },
  ru: { /* same structure, all strings in Russian */ }
};
```

**Escaping in highlights:** highlights are rendered with `{{{triple-stache}}}` (raw HTML), so special characters must be pre-escaped in the data: use `&amp;` for `&`, `&lt;` for `<`. All other fields use `{{double}}` (auto-escaped) — keep raw strings there.

## What to Edit Where

| Change needed | Edit file |
|---|---|
| Bio content (text, experience, skills) | `src/content-data.js` |
| SEO meta tags, OG descriptions | `build-meta.js` |
| HTML layout/structure | `src/partials/page.html` |
| Styles, animations, responsive | `src/styles.css` |
| Theme, scroll, sidebar logic | `src/app.js` |
| Avatar image | Replace `public/assets/avatar-source.png`, run `npm run build` |

## Key Notes

- `skills` is a **plain object** `{ categoryName: [...] }` — not an array
- `education` is a **single object** — not an array
- `languages[].lang` — the key is `lang`, not `name`
- `contacts` is a **plain object** — not an array
- Experience `highlights` — the key is `highlights`, not `bullets` or `tags`
- `dist/` is gitignored — built and deployed via GitHub Actions
